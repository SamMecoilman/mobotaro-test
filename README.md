# 🔄 開発方針アップデート note（2026-）

## 現在の判断
- 既存コードは開発途中のため **ロジックとしては信用しない**
- ただし、リポジトリ・履歴・設定・文脈は再利用する
- 「全部削除して0から」ではなく、**骨格を使って再出発する**

## 今後のゲーム方向性（再定義）
- **PvEオンリー**
- 同時接続最大100人を想定したオンライン構造
- 参考構造：Realm of the Mad God Exalt 型
- 戦闘の核はサバイバー系（Vampire Survivors ライク）

### Survivor Battle Area（定義）
- プレイ人数：2〜4人協力（PvE）
- セッション時間：5〜10分
- 操作：移動のみ（攻撃はオート）
- 敵が大量にスポーンし、時間経過で難易度上昇
- レベルアップ時にランダム強化を選択
- 当たり判定：2D（円形）
- サーバ権威（authoritative）
- 高頻度tick（15〜20/s）

## 世界観
- 学園バトルもの
- 放課後の校舎・廊下・教室・体育館などが舞台
- 日常空間が戦場として機能する設計

## ビジュアル方針
- 背景：HD-2D（オクトパストラベラー型の3Dジオラマ）
- キャラクター／敵／弾：ドット絵2Dスプライト
- ゲームロジック・当たり判定は完全2D
- 視認性最優先（戦闘中DOFなし、背景は低彩度）

## AI運用ルール
- ChatGPT：アイディア出し・設計・言語化
- Codex CLI：リポジトリを作業対象として実ファイル編集
- 人間：最終判断、commit / push のみ実行
- 既存ロジックを壊さない範囲で改変する
- 不要な新規ファイル増殖は避け、既存構造を拡張する
